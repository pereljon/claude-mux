סשנים מתמשכים של Claude Code לכל הפרויקטים שלך - נגישים מכל מקום דרך אפליקציית Claude לנייד. ***מנוהל על ידי Claude!***

## התקנה

```bash
curl -fsSL https://github.com/pereljon/claude-mux/releases/latest/download/install.sh | bash
```

לאחר מכן הפעל סשן:

```bash
claude-mux ~/path/to/your/project
```

תוכנית ההתקנה שואלת אם ברצונך להפעיל סשן home בעת הכניסה למערכת. אם תסכים, סשן Claude מוגן מופעל אוטומטית בכל פעם שאתה מתחבר - תמיד נגיש מהטלפון או מכל לקוח Remote Control, גם אם לעולם לא תפתח טרמינל.

זהו! אתה בסשן Claude מתמשך ומודע-סשן עם Remote Control מופעל. **מכאן, הכל בשיחה.**

[Homebrew, התקנה ידנית ואפשרויות נוספות](../docs/INSTALL.md)

## למה

Remote Control מבטיח Claude Code מכל מקום - אבל בלי ניהול סשנים, זה ממשק מדרגה שנייה גם מ-Claude Desktop:

- **סשנים מתים** כשסוגרים את הטרמינל
- **הקשר השיחה** לא מתחדש אוטומטית
- **אין בסיס פעולות** - שום דבר לא רץ כשאתה שולף את הטלפון, אלא אם השארת משהו פתוח
- **Remote Control דורש סשן פעיל** - אי אפשר להתחיל אחד מ-RC
- **פקודות slash לא עובדות בסשני RC** - אין החלפת מודל, דחיסה, או שינוי מצב הרשאות
- **התחלת פרויקטים חדשים** - דורשת יצירת תיקייה ידנית, אתחול git, כתיבת CLAUDE.md ובחירת מודל
- **אין ניהול פרויקטים** - אין דרך לראות פרויקטים לא פעילים, או לשנות שם, להעביר ולמחוק פרויקטים בלי לשבור היסטוריה

**claude-mux סוגר את הפער בניהול סשנים.** הוא עוטף את Claude Code ב-tmux כדי שסשנים ישרדו, מזריק system prompt כדי ש-Claude יוכל לנהל את הסשנים שלו, ומנתב פקודות slash דרך tmux כדי שיעבדו דרך Remote Control. ברגע שסשן רץ, אתה מנהל הכל בשיחה עם Claude - בטרמינל או באפליקציה לנייד.

## מה אפשר לעשות בסשן claude-mux

- **לנהל כל סשן מכל סשן** - להפעיל, לעצור, להפעיל מחדש, לרשום ולדחוס פרויקטים בשפה טבעית
- **לגשת לכל דבר מכל מקום** - לכל סשן מופעל Remote Control, כך שאפליקציית Claude לנייד, למחשב שולחני, או כל לקוח מרוחק הוא ממשק מלא
- **להחליף מודלים ומצבי הרשאות** - אמור "עבור ל-Haiku" או "עבור למצב plan" ו-Claude מטפל בזה, גם דרך Remote Control
- **ליצור פרויקטים חדשים** - "צור פרויקט חדש בשם my-app" מגדיר את התיקייה, git, CLAUDE.md, ומפעיל סשן. תבניות CLAUDE.md מאפשרות שימוש חוזר בהוראות בין פרויקטים.
- **לשמור סשנים חיים לאורך אתחולים** - סשן home אופציונלי מופעל בכניסה למערכת ונשאר פעיל; כל הסשנים ממשיכים אוטומטית את השיחה האחרונה
- **לשלוח פקודות slash דרך Remote Control** - Claude מנתב `/model`, `/compact`, `/clear` ופקודות slash נוספות לסשן הפעיל, עוקף [מגבלה ידועה](https://github.com/anthropics/claude-code/issues/30674)
- **לשמור היסטוריית שיחות** - שינוי שם, העברה והפעלה מחדש של פרויקטים שומרים אוטומטית על היסטוריית השיחה
- **לארגן פרויקטים** - להסתיר, לשנות שם, להעביר, למחוק ולהגן על פרויקטים מתוך כל סשן
- **תמיכה במספר חשבונות GitHub** - מזהה SSH aliases ב-`~/.ssh/config` ומזריק אותם לסשנים כדי ש-Claude ישתמש בחשבון הנכון לכל פרויקט
- **תמיכה במספר כלי CLI לקוד** - יוצר אוטומטית symlinks של `AGENTS.md` ו-`GEMINI.md` כדי ש-Codex CLI, Gemini CLI ואחרים ישתפו הוראות
- **עובד בכל שפה** - פקודות שיחה מזוהות לפי כוונה, לא לפי מילות מפתח

## לדבר עם Claude

ככה משתמשים ב-claude-mux יום-יום. כל סשן מוזרק עם פקודות כדי ש-Claude יוכל לנהל סשנים, להחליף מודלים, לשלוח פקודות slash וליצור פרויקטים חדשים - הכל מתוך השיחה. אין צורך לזכור דגלי CLI.

```
אתה: "status"
Claude: מדווח על שם הסשן, מודל, מצב הרשאות, שימוש בהקשר, ורושם את כל הסשנים

אתה: "הצג סשנים פעילים"
Claude: מציג את כל הסשנים הפעילים עם הסטטוס שלהם

אתה: "הפעל סשן לפרויקט api-server שלי"
Claude: מפעיל סשן ב-~/Claude/work/api-server

אתה: "צור פרויקט חדש בשם mobile-app עם תבנית web"
Claude: יוצר את תיקיית הפרויקט, מאתחל git, מחיל את התבנית, מפעיל סשן

אתה: "העבר את הסשן הזה ל-Haiku"
Claude: שולח /model haiku לעצמו דרך tmux

אתה: "דחוס את סשן api-server"
Claude: שולח /compact לסשן api-server

אתה: "הפעל מחדש את סשן web-dashboard"
Claude: מכבה ומפעיל מחדש את הסשן, שומר על הקשר השיחה

אתה: "העבר את סשן api-server למצב plan"
Claude: מפעיל מחדש את הסשן עם מצב הרשאות plan

אתה: "העבר את הסשן הזה למצב yolo"
Claude: עובר למצב bypassPermissions דרך Shift+Tab - בלי צורך בהפעלה מחדש

אתה: "באיזה מצב הסשן הזה"
Claude: מדווח על מצב ההרשאות הנוכחי (default, acceptEdits, plan, bypassPermissions)

אתה: "העבר את הסשן הזה ל-Opus"
Claude: שולח /model opus לעצמו דרך tmux

אתה: "נקה את הסשן הזה"
Claude: שולח /clear לעצמו, מאפס את השיחה

אתה: "הסתר את הפרויקט הזה"
Claude: כותב .claudemux-ignore כדי שהפרויקט לא יופיע ברשימות -L

אתה: "הגן על הסשן הזה"
Claude: כותב .claudemux-protected ומגדיר את סימון tmux - כיבוי דורש עכשיו --force

אתה: "האם הסשן הזה מוגן"
Claude: בודק אם קיים .claudemux-protected בתיקיית הפרויקט ומדווח

אתה: "מחק את פרויקט old-prototype"
Claude: מאשר בצ'אט, ואז מעביר את תיקיית הפרויקט לפח המערכת

אתה: "שנה את שם הפרויקט הזה ל-my-new-name"
Claude: עוצר את הסשן, משנה שם תיקייה, מעביר היסטוריית שיחות, מפעיל מחדש

אתה: "שמור את זה כתבנית בשם web"
Claude: מעתיק CLAUDE.md ל-~/.claude-mux/templates/web.md

אתה: "tip"
Claude: מציג טיפ - אותו טיפ כל היום, או אקראי אם TIP_MODE=random מוגדר

אתה: "הפעל טיפים" / "כבה טיפים"
Claude: מפעיל או מכבה את הטיפ היומי בכל הפרויקטים

אתה: "עדכן claude-mux"
Claude: מזהיר שכל הסשנים יופעלו מחדש, מבקש אישור, ואז מעדכן ומפעיל מחדש

אתה: "עצור את כל הסשנים"
Claude: סוגר בצורה מסודרת את כל הסשנים המנוהלים

אתה: "help"
Claude: מציג את רשימת הפקודות המלאה
```

**פקודות אלה עובדות בכל שפה.** אם תקליד את המקבילה בעברית, ביפנית, בספרדית או בכל שפה אחרת, Claude מזהה את הכוונה ומריץ את הפקודה המתאימה.

**הקלד `help` בתוך כל סשן כדי לראות את רשימת הפקודות המלאה.**
