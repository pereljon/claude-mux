## Что такое claude-mux?

Шелл-скрипт, оборачивающий Claude Code в tmux для постоянных сессий. Сессии переживают закрытие терминала, возобновляют контекст разговора при перезапуске и доступны из мобильного приложения Claude через Remote Control. Вы управляете всем, разговаривая с Claude внутри сессии.

## Работает ли он на Linux?

Пока нет. Только macOS (Apple Silicon и Intel). Поддержка Linux запланирована на v2.0. Установщик запускается на Linux, но пропускает настройку LaunchAgent и выводит уведомление. Сам бинарный файл работает, но механизма автозапуска (systemd-сервиса или аналога) пока нет.

## Что такое home-сессия?

Home-сессия это универсальная сессия Claude, расположенная в вашем базовом каталоге (`~/Claude` по умолчанию). Если `LAUNCHAGENT_MODE=home` (по умолчанию), она запускается автоматически при входе в систему и работает весь день. Она **защищена** по умолчанию, то есть `--shutdown home` отказывается её останавливать без `--force`.

Используйте home-сессию как всегда доступную точку входа из мобильного приложения Claude. Оттуда можно просматривать проекты, запускать другие сессии, управлять конфигурацией и выполнять общие задачи, не привязанные к конкретному проекту.

## Что такое Remote Control?

Remote Control (RC) это функция Claude Code, позволяющая подключаться к запущенной сессии Claude из мобильного приложения Claude или Claude Desktop. claude-mux запускает каждую сессию с включённым `--remote-control`, поэтому все сессии автоматически появляются в списке RC. После подключения вы общаетесь с Claude так же, как в терминале. claude-mux также обходит ограничения RC, такие как неработающие слэш-команды, маршрутизируя их через tmux.

## Что такое режимы разрешений?

Claude Code имеет четыре режима разрешений, определяющих степень автономии Claude:

| Режим | Поведение |
|-------|-----------|
| `default` | Claude спрашивает перед выполнением команд или редактированием файлов |
| `acceptEdits` | Claude автоматически применяет правки файлов, но спрашивает перед шелл-командами |
| `plan` | Claude может только читать и планировать, без записи и выполнения команд |
| `bypassPermissions` | Claude выполняет всё без запроса (требуется подтверждение при первом запуске) |

Установите режим по умолчанию для всех проектов через `DEFAULT_PERMISSION_MODE` в конфигурации. Переключите режим запущенной сессии, сказав "switch this session to plan mode" (или любое другое имя режима). "yolo" это псевдоним для `bypassPermissions`.

Переключение на `bypassPermissions` из другого режима использует навигацию Shift+Tab и не требует перезапуска. Переключение с `bypassPermissions` на другой режим требует перезапуска, который claude-mux выполняет автоматически.

## Как сбросить сессию?

Три варианта, в зависимости от задачи:

- **Clear** ("clear this session"): отправляет `/clear` в сессию. Очищает историю разговора и начинает заново. Сессия продолжает работать.
- **Compact** ("compact this session"): отправляет `/compact` в сессию. Суммирует разговор в более короткий контекст, освобождая окно контекста. История сохраняется в сжатом виде.
- **Restart** ("restart this session"): завершает Claude и перезапускает с `claude -c`, что возобновляет последний разговор. Используйте, когда нужен чистый процесс (например, после изменения режима разрешений или когда Claude застрял).

## Что такое шаблоны?

Шаблоны это переиспользуемые файлы CLAUDE.md, хранящиеся в `~/.claude-mux/templates/`. При создании нового проекта через `-n` применяется шаблон по умолчанию (или указанный через `--template NAME`), который копируется в проект как его CLAUDE.md.

Создать шаблон: "save this as a template named web" (копирует CLAUDE.md текущего проекта в `~/.claude-mux/templates/web.md`).

Использовать шаблон: `claude-mux -n ~/projects/my-app --template web` или изнутри сессии: "create a new project called my-app using the web template".

Список шаблонов: "list templates" или `claude-mux --list-templates`.

## Как работает совет дня?

Claude Code Stop-хук в `.claude/settings.local.json` каждого проекта вызывает `claude-mux --tipotd` после каждого хода разговора. Команда проверяет, был ли совет уже показан сегодня (через `~/.claude-mux/.tip-date`). Если да, завершается примерно за 6 мс. Если нет, выводит совет и записывает сегодняшнюю дату.

Советы включены по умолчанию (`TIP_OF_DAY=true`). Переключайте через "enable tips" или "disable tips" внутри любой сессии. `TIP_MODE=daily` показывает один и тот же совет весь день; `TIP_MODE=random` выбирает случайный совет при каждом вызове (со Stop-хуком это означает один случайный совет в день благодаря ежедневному шлюзу).

Команда `--tip` всегда работает вне зависимости от ежедневного шлюза, так что можно сказать "tip" в любое время.

## Можно ли использовать с несколькими аккаунтами GitHub?

Да. claude-mux обнаруживает записи `Host github.com-*` в `~/.ssh/config` и внедряет их в системный промпт каждой сессии. Claude знает, какие SSH-алиасы доступны, и может использовать нужный при настройке git-remote.

Пример настройки `~/.ssh/config`:

```
Host github.com-work
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_work

Host github.com-personal
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_personal
```

Claude будет использовать `git@github.com-work:org/repo.git` для рабочих репозиториев и `git@github.com-personal:user/repo.git` для личных.

## Где хранится состояние?

| Расположение | Содержимое |
|--------------|------------|
| `~/.claude-mux/config` | Пользовательская конфигурация (подключается как bash) |
| `~/.claude-mux/templates/` | Файлы шаблонов CLAUDE.md |
| `~/.claude-mux/.tip-date` | Дата последнего показанного совета |
| `~/.claude-mux/.update-check` | Кэшированный результат проверки обновлений |
| `~/Library/Logs/claude-mux.log` | Файл лога (настраивается через `LOG_DIR`) |
| `~/Library/LaunchAgents/com.user.claude-mux.plist` | plist LaunchAgent (генерируется `--install`) |
| `.claudemux-protected` (в каждом проекте) | Помечает сессию как защищённую от остановки |
| `.claudemux-ignore` (в каждом проекте) | Скрывает проект из списков |

Файлы-маркеры (`.claudemux-*`) расположены в корне каждого проекта и следуют за папкой при переименованиях, перемещениях и синхронизациях. Они автоматически добавляются в `.gitignore`.

История разговоров управляется самим Claude Code и хранится в `~/.claude/projects/`.

## Что будет с автообновлением, если я форкну claude-mux?

Проверка обновлений и команда `--update` жёстко привязаны к репозиторию `pereljon/claude-mux` на GitHub. Если вы форкнете, проверка обновлений по-прежнему будет сравнивать с апстрим-релизом, а `--update` перезапишет бинарный файл вашего форка апстримом. Установите `UPDATE_CHECK=false` в `~/.claude-mux/config`, чтобы отключить, или измените URL репозитория в функциях `check_for_update()` и `do_update()` в скрипте.

## Как установить через Homebrew?

```bash
brew tap pereljon/tap
brew install claude-mux
claude-mux --install
```

Обновление: `brew upgrade claude-mux`. Примечание: если установка выполнена через Homebrew, `--update` автоматически делегирует команду `brew upgrade`.

## Чем это отличается от `claude --worktree --tmux`?

`claude --worktree --tmux` создаёт сессию tmux для изолированного git worktree, предназначенного для параллельных задач кодирования. claude-mux управляет постоянными сессиями для ваших настоящих каталогов проектов, с включённым Remote Control, внедрением системного промпта для самоуправления, возобновлением разговоров и управлением жизненным циклом сессий. Они решают разные задачи.

## Почему сессии показывают "Not logged in"?

Это происходит при первом запуске, если связка ключей macOS заблокирована, что часто бывает, когда LaunchAgent стартует до разблокировки связки ключей после входа. Решение: запустите `security unlock-keychain` в обычном терминале, затем подключитесь к любой сессии (`claude-mux -t <name>`) и выполните `/login` для прохождения аутентификации в браузере. После этого перезапустите все сессии: они подхватят сохранённые учётные данные.

## Можно ли подключить несколько терминалов к одной сессии?

Да. Это стандартное поведение tmux. Запуск `claude-mux` в каталоге, где уже есть запущенная сессия, подключает к ней. Несколько терминалов видят содержимое одной сессии в реальном времени.

## Как остановить home-сессию навсегда?

LaunchAgent имеет `KeepAlive: true`, поэтому завершение home-сессии вызовет перезапуск примерно через 60 секунд. Чтобы остановить навсегда, отключите LaunchAgent:

```bash
claude-mux --install --launchagent-mode none
```

## Что означает сообщение "Session ready!"?

Когда сессия запускается или перезапускается, claude-mux отправляет промпт `Ready?` после завершения загрузки Claude. Внедрённый промпт указывает Claude ответить "Session ready!" и ничем больше. Это подтверждает, что сессия жива и внедрение системного промпта работает. Можно проигнорировать.

## Как скрыть проект из списков?

Скажите "hide this project" внутри любой сессии или выполните `claude-mux --hide my-project`. Это создаёт файл-маркер `.claudemux-ignore`. Проект не будет отображаться в выводе `claude-mux -L`. Чтобы увидеть скрытые проекты: `claude-mux -L --hidden`. Чтобы показать снова: "show this project" или `claude-mux --show my-project`.

## Как удалить claude-mux?

```bash
claude-mux --uninstall
```

Это удаляет хуки советов и правила разрешений из всех проектов, выгружает LaunchAgent и при желании удаляет `~/.claude-mux/`. Команда сообщает путь к бинарному файлу, чтобы вы могли удалить его вручную (или `brew uninstall claude-mux` при установке через Homebrew).

## Работают ли слэш-команды через Remote Control?

Не нативно. Claude Code не поддерживает слэш-команды (`/model`, `/clear` и др.) в RC-сессиях. claude-mux обходит это, внедряя в каждую сессию `claude-mux -s`, чтобы Claude мог отправлять слэш-команды самому себе через tmux. Просто скажите "switch to Haiku" или "compact this session", и Claude всё сделает.

## Не могу выделить текст в сессии

Удерживайте **Option** (macOS) или **Shift** (Linux/Windows-терминалы) при клике и перетаскивании. Это обходит захват мыши tmux и копирует выделение в системный буфер обмена. Менять настройки не нужно.

## Какие языки поддерживаются для диалоговых команд?

Все. Триггерные фразы ("help", "status", "list sessions" и т.д.) работают на любом языке. Claude определяет намерение из естественного языка пользователя и выполняет соответствующую команду. README также переведён на 12 языков.
